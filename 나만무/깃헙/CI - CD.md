## CI/CD 파이프라인

### A. Feature 브랜치 Push (지속적 통합)

Feature 브랜치에 push할 때마다 다음 자동 검사가 실행됩니다:

**a.1** Lint 검사 - 코드 포맷팅 및 스타일 검증  
**a.2** Unit 테스트 - 개별 컴포넌트 테스트  
**a.3** Build 확인 - 코드가 정상적으로 컴파일되는지 확인

**목적:** 개발 초기 단계에서 오류를 빠르게 발견  
**실행 시간:** ~3분

### B. Develop 브랜치 Push (스테이징 배포)

코드가 develop으로 병합될 때 (일반적으로 목요일), 다음의 포괄적인 검사가 실행됩니다:

**b.1** Lint - 코드 포맷팅 및 스타일 검증  
**b.2** 유닛 테스트 - 개별 컴포넌트 테스트  
**b.3** 통합 테스트 - 컴포넌트 간 상호작용 테스트  
**b.4** E2E 테스트 - 종단간 사용자 시나리오 테스트  
**b.5** 빌드 - 프로덕션 배포용 아티팩트 생성  
**b.6** 스테이징 환경 배포 - staging.damso.app에 자동 배포

**목적:** 데모 준비 전 종합적인 검증  
**실행 시간:** ~10분  
**결과:** 팀 테스트를 위해 스테이징 환경에 코드 배포

### C. Main 브랜치 Push (프로덕션 배포)

Develop이 main으로 병합될 때 (금요일 오전, 데모용):

**c.1** 전체 테스트 스위트 (lint + unit + integration + E2E)  
**c.2** 빌드  
**c.3** 프로덕션 배포 - damso.app에 배포  
**c.4** Smoke 테스트 - 핵심 기능 동작 확인
